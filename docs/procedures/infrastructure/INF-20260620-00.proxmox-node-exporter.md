# INF-20260620-00 — Install node-exporter on Proxmox VE host

## Context

The `pve-exporter` LXC ships LXC-level metrics (CPU, memory, network, filesystem
of the container itself) via `catalog.lxcAgent.nodeExporter`. That only covers the
LXC. Hypervisor-level metrics — host disk I/O, hardware sensors, NVMe health,
host filesystem usage — require node-exporter running directly on the PVE host.

This procedure installs and configures `prometheus-node-exporter` on a Proxmox VE
node so the `pve-exporter` LXC can scrape it.

## Prerequisites

* SSH access to the PVE host as root
* Proxmox host: `pve-01.pve.chezmoi.sh` (10.0.0.11)
* `pve-exporter` LXC reachable on the Proxmox bridge (`vmbr0`, 10.0.0.x range)

## Procedure

### 1. Install

Proxmox runs Debian. The package is in the default repositories:

```sh
apt-get update
apt-get install -y prometheus-node-exporter
```

### 2. Configure

Edit `/etc/default/prometheus-node-exporter` to restrict the listen address to
the Proxmox management bridge and enable the full collector set suitable for
bare-metal:

```sh
cat > /etc/default/prometheus-node-exporter << 'EOF'
# Listen only on the Proxmox management bridge — not exposed publicly.
# The pve-exporter LXC scrapes this from its side of vmbr0.
ARGS="--web.listen-address=10.0.0.11:9100 \
  --collector.systemd \
  --collector.processes \
  --collector.filesystem.mount-points-exclude='^/(dev|proc|run/credentials/.+|sys|var/lib/docker/.+|var/lib/containers/.+)($|/)' \
  --collector.diskstats.device-exclude='^(ram|loop|fd|(h|s|v|xv)d[a-z]|nvme[0-9]+n[0-9]+p)[0-9]+$'"
EOF
```

> The `filesystem` and `diskstats` exclusion patterns strip ephemeral mounts
> (proc, sysfs, loop devices, partition slices) — keeping only meaningful storage
> devices and mount points.

### 3. Enable and start

```sh
systemctl enable prometheus-node-exporter
systemctl restart prometheus-node-exporter
systemctl status prometheus-node-exporter
```

Verify it responds:

```sh
curl -s http://10.0.0.11:9100/metrics | grep node_uname_info
```

### 4. PVE firewall (if enabled)

If the PVE datacenter or node firewall is active, allow inbound TCP 9100 from
the LXC network:

```sh
# In /etc/pve/nodes/pve-01/host.fw, add under [RULES]:
IN ACCEPT --source 10.0.0.0/24 --dport 9100 -p tcp -log nolog
```

Or via the Proxmox web UI: Datacenter → pve-01 → Firewall → Add rule.

### 5. Wire up the scrape target in pve-exporter

In `projects/chezmoi.sh/src/infrastructure/proxmox/lxc/pve-exporter/modules/o11y.nix`,
add a scrape target for the PVE host node-exporter alongside the existing
`pve_exporter` job:

```nix
metrics = {
  enable = true;
  scrapeTargets = [
    {
      jobName = "pve_exporter";
      targets = [ "127.0.0.1:9221/pve?target=${pveHost}&cluster=1&node=1" ];
    }
    {
      jobName = "node_exporter_pve";
      targets = [ "10.0.0.11:9100" ];
    }
  ];
};
```

Then rebuild and upgrade the pve-exporter LXC:

```sh
cd projects/chezmoi.sh/src/infrastructure/proxmox/lxc/pve-exporter
# bump version in flake.nix
mise run lxc:build
mise run lxc:push -- pve-01.pve.chezmoi.sh
mise run lxc:upgrade -- pve-01.pve.chezmoi.sh
```

## Verification

After the pve-exporter LXC is upgraded, check that the new job appears in
VictoriaMetrics:

```promql
# Host filesystem usage
node_filesystem_avail_bytes{job="node_exporter_pve", fstype!="tmpfs"}

# Host disk I/O
rate(node_disk_read_bytes_total{job="node_exporter_pve"}[5m])

# Host CPU
rate(node_cpu_seconds_total{job="node_exporter_pve", mode="idle"}[1m])

# NVMe health (if available)
node_nvme_info{job="node_exporter_pve"}
```
